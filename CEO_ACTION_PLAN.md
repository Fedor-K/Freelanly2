## Система принятия решений

### Framework: ICE Score

Каждая задача оценивается по:
- **I**mpact (1-10): Влияние на MRR
- **C**onfidence (1-10): Уверенность в успехе
- **E**ase (1-10): Простота реализации

**ICE Score = (I + C + E) / 3**

Приоритет задачам с высоким ICE.

### Правила принятия решений

| Ситуация | Действие |
|----------|----------|
| MRR не растёт | Анализ воронки, A/B тесты |
| Высокий churn (>5%) | Опросы пользователей, улучшение onboarding |
| Низкая конверсия (<2%) | Пересмотр pricing, value proposition |
| Мало трафика | SEO оптимизация, content marketing |
| Мало вакансий | Добавление источников данных |

## Ключевые метрики (North Star)

### Primary Metric
**MRR (Monthly Recurring Revenue)** — единственная метрика, которая определяет успех

### Secondary Metrics (влияют на MRR)

| Метрика | Формула | Цель |
|---------|---------|------|
| Conversion Rate | Paid Users / Total Users | >3% |
| Churn Rate | Churned / Active Paid | <5% |
| ARPU | MRR / Paid Users | $19+ |
| LTV | ARPU / Churn Rate | $380+ |
| CAC | Marketing Spend / New Customers | <$50 |
| LTV/CAC | LTV / CAC | >3 |

### Leading Indicators (предсказывают рост)

| Метрика | Почему важна |
|---------|--------------|
| Daily Active Users | Больше DAU → больше конверсий |
| Job Views | Показывает ценность контента |
| Job Applications | Engagement с продуктом |
| Email Signups | Потенциальные платящие |

## Критические риски и митигация

### Риск 1: Низкая конверсия в Pro
**Вероятность:** Высокая
**Митигация:**
- Тестировать разные ценовые точки ($9, $19, $29)
- Добавить trial period
- Улучшить value proposition
- Social proof (testimonials)

### Риск 2: Технические проблемы
**Вероятность:** Средняя
**Митигация:**
- Мониторинг ошибок (Sentry)
- Автоматические алерты
- Rollback процедуры

### Риск 3: Конкуренция
**Вероятность:** Средняя
**Митигация:**
- Фокус на нише (remote jobs)
- Уникальные фичи (Salary Insights)
- Быстрое развитие

### Риск 4: Зависимость от LinkedIn/Apify
**Вероятность:** Низкая
**Митигация:**
- Диверсификация источников
- Прямые интеграции с ATS
- Собственная база компаний

## Workflow: Как я буду работать

### При каждом запросе:

1. **Смотрю на цель** — приближает ли задача к $10K MRR?
2. **Оцениваю приоритет** — ICE score
3. **Проверяю зависимости** — что блокирует?
4. **Выполняю** — с фокусом на качество
5. **Измеряю** — как повлияло на метрики?

### При принятии решений:

```
IF задача не влияет на MRR:
  → Отклонить или отложить

IF задача влияет, но низкий ICE:
  → Отложить до завершения высокоприоритетных

IF задача влияет и высокий ICE:
  → Выполнить немедленно
```
